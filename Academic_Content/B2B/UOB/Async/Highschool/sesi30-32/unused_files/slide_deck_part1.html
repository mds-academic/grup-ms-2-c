Apa Itu Input? 📥

Input adalah semua data atau informasi yang dimasukkan oleh pengguna (user) ke dalam program kita.

Di Python, kita menggunakan perintah input() untuk menerima data dari keyboard:

name = input("Nama transaksi: ")
amount = input("Nominal transaksi: ")
transaction_type = input("Jenis transaksi: ")

Input membuat program kita menjadi interaktif dan hidup! Tapi, input juga bisa jadi sumber masalah utama jika pengguna memasukkan data asal-asalan. ⚠️

Input Pengguna Tidak Selalu Benar! 🤯

Bayangkan kamu membuat program keuangan, lalu pengguna memasukkan data-data aneh seperti ini:

Apa yang Diketik Pengguna Masalahnya Apa?
"" (Langsung Enter) Input kosong, nama transaksi hilang!
"abc" Program minta angka nominal, tapi diisi huruf.
-50000 Uang tidak boleh minus (negatif) dalam input biasa.
" pengeluaran " Ada spasi berlebih di awal dan akhir teks.
"PEMASUKAN" Huruf kapital semua, bisa membingungkan komputer.

Program yang cerdas harus bisa mengecek dan membersihkan data kotor tersebut! 🧹

Apa Itu Validasi Input? 🚦

Validasi input adalah proses pemeriksaan untuk memastikan bahwa data yang dimasukkan pengguna sudah benar, aman, dan masuk akal sebelum diproses program.

Golden Rule: "Jangan pernah langsung percaya pada data dari pengguna!" 🛡️

Hal-hal yang perlu divalidasi oleh program keuangan kita:

  • Apakah input kosong? (Nama transaksi harus ada).
  • Apakah berupa angka? (Nominal harus numerik).
  • Apakah angkanya positif? (Nominal transaksi > 0).
  • Apakah kategori sesuai pilihan? (Hanya "pemasukan" atau "pengeluaran").

Validasi Input Bagaikan Petugas Penjaga 👮‍♂️

Bayangkan kamu ingin bermain di Dufan atau wahana roller coaster. Sebelum masuk, petugas akan memeriksa:

🎟️ Pemeriksaan 1:
Apakah kamu memegang tiket resmi? (Bukan tiket palsu/kosong).
📏 Pemeriksaan 2:
Apakah tinggi badanmu aman untuk menaiki wahana? (Tidak terlalu pendek).

Jika salah satu syarat gagal, kamu tidak boleh masuk. Program komputer pun sama! Jika data tidak lolos validasi, program akan menolak memprosesnya demi **keamanan data**. 🎢

Apa Itu Sanitasi Input? 🧼

Selain memeriksa (validasi), kita juga bisa merapikan data yang diketik pengguna agar seragam. Ini disebut dengan Sanitasi Input.

Misalnya, pengguna mengetik jenis transaksi dengan spasi acak atau huruf besar-kecil campuran:

Input Kotor: " PeMaSuKaN " → Dibersihkan → Input Bersih: "pemasukan"

Di Python, kita punya function bawaan string yang sangat praktis:

  • .strip() untuk menghapus spasi tak berguna di awal dan akhir teks.
  • .lower() untuk mengubah semua huruf menjadi kecil (lowercase).

Membersihkan Spasi dengan `.strip()` 🧹

Terkadang, spasi ekstra yang tidak sengaja tertekan bisa membuat pencarian program error. Gunakan .strip() untuk membuang spasi di ujung kiri dan kanan.

name = input("Nama: ").strip()

Validasi Input Kosong 📭

Bagaimana jika pengguna terburu-buru dan langsung menekan Enter tanpa menulis nama transaksi?

Kita bisa mengecek apakah variabel tersebut bernilai string kosong "":

name = input("Nama transaksi: ").strip()

if name == "":
    print("Nama transaksi tidak boleh kosong.")
else:
    print("Nama transaksi:", name)

Dengan cara ini, program kita tidak akan menyimpan data hantu atau nama yang kosong! 👻

Validasi Jenis Transaksi 🔄

Dalam pencatatan keuangan, jenis transaksi harus dibatasi agar komputer tidak bingung saat menghitung total saldo akhir.

Kita hanya memperbolehkan dua jenis: "pemasukan" atau "pengeluaran".

transaction_type = input("Jenis: ").strip().lower()

if transaction_type == "pemasukan" or transaction_type == "pengeluaran":
    print("Jenis transaksi valid!")
else:
    print("Jenis harus pemasukan atau pengeluaran!")

Kenapa Validasi Penting di Program Keuangan? 🏦

Program keuangan mengelola data yang sangat sensitif, yaitu Uang! 💵

Jika validasi tidak ketat, hal aneh bisa terjadi:

Pemasukan diisi angka negatif: amount = -50000

Secara matematika, memasukkan pengeluaran dengan angka negatif malah bisa **menambah** saldo akhir karena minus bertemu minus!

Ini adalah celah (bug/exploit) berbahaya yang sering dimanfaatkan orang jika program tidak divalidasi. 🛡️

Validasi Nominal: Harus Lebih dari Nol! 💸

Untuk mengamankan perhitungan uang, pastikan nominal transaksi selalu lebih besar dari 0.

amount = int(input("Nominal transaksi: "))

if amount <= 0:
    print("Nominal harus lebih dari 0!")
else:
    print("Nominal valid:", amount)

Angka nominal seperti Rp0 atau minus tidak masuk akal dalam catatan keuangan riil.

Membuat Function `clean_text()` 🛠️

Daripada menulis .strip().lower() berulang-ulang untuk setiap input teks, kita bisa membungkusnya menjadi satu function yang rapi (reusable code):

def clean_text(text):
    return text.strip().lower()

category = clean_text(" PEMASUKAN ")
print(category) # Output: pemasukan

Sekarang, membersihkan teks jadi super gampang!

Membuat Function `validate_type()` 🔍

Selanjutnya, mari kita buat function untuk memvalidasi jenis transaksi. Function ini akan mengembalikan nilai Boolean (True atau False):

def validate_type(transaction_type):
    transaction_type = clean_text(transaction_type)
    if transaction_type == "pemasukan" or transaction_type == "pengeluaran":
        return True
    else:
        return False

Membuat Function `validate_amount()` ⚖️

Kita juga buat function khusus untuk memvalidasi nominal uang agar bernilai positif:

def validate_amount(amount):
    if amount > 0:
        return True
    else:
        return False

Contoh penggunaan:

print(validate_amount(50000)) # Output: True
print(validate_amount(-10000)) # Output: False

Error Handling dengan `try-except`

Menjaga Program Agar Tetap Hidup Saat Badai Error Datang!

Jenis Error yang Sering Muncul 🐛

Error (exception) di Python terjadi ketika kode melanggar aturan sistem saat berjalan.

Tipe Error Penyebabnya
ValueError Tipe data tidak bisa diubah secara paksa (misal: huruf diubah ke int).
TypeError Melakukan operasi pada dua tipe data yang tidak cocok (misal: string + int).
ZeroDivisionError Membagi suatu angka dengan angka 0.

Contoh Kasus: ValueError 💥

Terjadi saat program meminta angka nominal, tapi diisi huruf "lima ribu":

amount = int(input("Nominal: "))

🧪 Coba Masukkan Huruf (Simulasi ValueError):

Ketik kata sembarang (misal: "seribu") dan lihat crash program:

Contoh Kasus: TypeError 💥

Terjadi saat kita mencoba menjumlahkan string dengan angka secara langsung:

# ❌ Ini ERROR:
amount = 50000
message = "Nominal transaksi: " + amount

Solusi yang benar: Ubah angka menjadi teks (string) terlebih dahulu menggunakan str():

# ✅ Ini BENAR:
amount = 50000
message = "Nominal transaksi: " + str(amount)

Contoh Kasus: ZeroDivisionError 💥

Terjadi ketika kamu membagi suatu angka dengan angka 0:

total_money = 100000
people = 0
result = total_money / people

🧪 Bagi Uang ke Orang (Simulasi ZeroDivisionError):

Mengenal Penyelamat: `try-except` 🛡️

Di Python, kita bisa menangani error secara elegan dengan try-except.

Sederhananya, jalurnya seperti ini:

"Coba jalankan kode ini. Jika terjadi error di tengah jalan, jangan hentikan program! Segera jalankan instruksi cadangan."
try:
    # Kode yang dicurigai bisa error
except:
    # Kode yang dijalankan jika benar-benar error

Analogi `try-except`: Rencana Cadangan 🚪

Bayangkan kamu sedang memegang kunci dan ingin membuka sebuah pintu:

🔑 Langkah `try`:
Kamu mencoba memutar kunci untuk membuka pintu.
🚪 Langkah `except`:
Jika kunci macet (error), kamu tidak menabrak pintu! Kamu menggunakan rencana cadangan (mengetuk pintu / meminta tolong).

Dengan try-except, program kita tidak akan langsung mati (close) di depan pengguna, melainkan tetap berjalan dengan pesan alternatif yang ramah. 😊

Menangani Input Huruf Saat Diminta Angka 🔢🛡️

Mari gunakan try-except untuk menangkap ValueError ketika user menginput huruf di kolom nominal:

try:
    amount = int(input("Nominal transaksi: "))
    print("Nominal:", amount)
except ValueError:
    print("Nominal harus berupa angka.")

Jika input berupa huruf, Python langsung melompat ke bagian except ValueError:. Program tetap selamat dan menampilkan pesan ramah! 🎉

Menangani Beberapa Error Sekaligus 🚦

Sebuah blok try bisa diikuti oleh beberapa except untuk menangani berbagai jenis kesalahan:

try:
    total = 100000
    people = int(input("Jumlah orang: "))
    result = total / people
    print("Masing-masing:", result)
except ValueError:
    print("Jumlah orang harus angka!")
except ZeroDivisionError:
    print("Jumlah orang tidak boleh 0!")

Debugging Bagaikan Memperbaiki Lampu Mati 💡🛠️

Debugging dengan Banyak Function ⚙️

Dalam program yang besar, kita biasanya membagi kode menjadi beberapa function yang bekerja sama:

def get_total(price, quantity):
    return price * quantity

def get_discount(total):
    return total * 0.1 # Diskon 10%

def get_final_price(total, discount):
    return total - discount

⚙️ Uji Alur Penghitungan Function:

Apa Itu Dependency Antar Function? 🔗

Dependency artinya suatu fungsi bergantung pada fungsi lainnya untuk mendapatkan input data.

total = get_total(price, quantity)
discount = get_discount(total) # Butuh hasil dari get_total
final_price = get_final_price(total, discount) # Butuh total dan discount

Jika fungsi pertama (get_total) menghasilkan nilai yang salah karena bug, maka fungsi-fungsi selanjutnya otomatis akan menghasilkan nilai yang salah pula! 💥

Mencari bug pada rantai ketergantungan ini membutuhkan ketelitian ekstra.

Cara Debugging Step-by-Step 🕵️‍♂️

Saat programmu menghasilkan nilai yang salah atau crash, ikuti 7 langkah logis berikut:

  1. 1. Baca pesan error yang muncul di terminal.
  2. 2. Cari baris kode yang disebut oleh pesan error tersebut.
  3. 3. Cek input data dan tipe datanya (apakah sesuai harapan?).
  4. 4. Tambahkan print() sementara untuk mengintip isi variabel.
  5. 5. Cek function satu per satu mulai dari function paling awal.
  6. 6. Perbaiki bagian logika/operator yang salah.
  7. 7. Jalankan ulang program untuk memastikan bug telah hilang.

Debugging Menggunakan `print()` 🖨️

Kita bisa menyelipkan perintah print() sementara di dalam function untuk melihat apakah data yang diterima sudah benar:

def get_total(price, quantity):
    print("DEBUG price:", price)
    print("DEBUG quantity:", quantity)
    return price * quantity

Contoh Bug pada Function 🐛

Perhatikan fungsi penghitung total belanja berikut:

def get_total(price, quantity):

Apa Itu Dictionary di Python? 📖🔑

Dictionary adalah wadah penyimpanan data yang menggunakan pasangan Key (Label) dan Value (Isi Data).

Dictionary sangat cocok untuk mencatat detail data suatu transaksi!

🔑 Key: "name"
📄 Value: "Beli buku"
🔑 Key: "type"
📄 Value: "pengeluaran"
🔑 Key: "amount"
📄 Value: 25000

transactions = []

def clean_text(text):
    return text.strip().lower()

def validate_type(transaction_type):
    transaction_type = clean_text(transaction_type)
    return transaction_type == "pemasukan" or transaction_type == "pengeluaran"

def validate_amount(amount):
    return amount > 0

def add_transaction(name, transaction_type, amount):
    transaction = {"name": name, "type": transaction_type, "amount": amount}
    transactions.append(transaction)

def summarize_transactions():
    income = sum(t["amount"] for t in transactions if t["type"] == "pemasukan")
    expense = sum(t["amount"] for t in transactions if t["type"] == "pengeluaran")
    print("Total Pemasukan:", income)
    print("Total Pengeluaran:", expense)
    print("Saldo:", income - expense)

try:
    name = input("Nama transaksi: ").strip()
    t_type = clean_text(input("Jenis transaksi (pemasukan/pengeluaran): "))
    amount = int(input("Nominal: "))

    if name == "":
        print("Nama tidak boleh kosong.")
    elif not validate_type(t_type):
        print("Jenis tidak valid.")
    elif not validate_amount(amount):
        print("Nominal harus lebih dari 0.")
    else:
        add_transaction(name, t_type, amount)
        print("Berhasil ditambahkan.")
except ValueError:
    print("Nominal harus berupa angka.")
