Materi Bacaan 01

Conditional: cara program mengambil keputusan

Baca bagian ini setelah menonton video untuk mengulang konsep kondisi, hasil True atau False, dan hubungan antara syarat dengan aksi.

Dasar konsep
A

Program tidak selalu berjalan lurus

Conditional membuat program memilih aksi yang berbeda berdasarkan keadaan atau data yang sedang diperiksa.

B

Kondisi adalah sebuah pertanyaan

Contohnya: “Apakah hujan?”, “Apakah umur minimal 17?”, atau “Apakah koin sudah 100?”.

C

Jawabannya hanya dua

Setiap kondisi menghasilkan True saat syarat terpenuhi atau False saat syarat tidak terpenuhi.

Dari keputusan sehari-hari ke kode

  • Jika hujan, bawa payung.
  • Jika baterai hampir habis, isi daya.
  • Jika pemain mendapat apel, tambah nyawa.
# Komputer mengecek syarat sebelum menjalankan aksi hujan = True if hujan: print("Bawa payung")

Intinya: kondisi bekerja seperti penjaga pintu kastil. Aksi di dalam blok hanya boleh berjalan ketika syaratnya benar (kamu memiliki kuncinya).

Analogi Penjaga Pintu 🏰

  • Syarat / Kondisi = Apakah kamu punya kuncinya?
  • True = Kamu punya kunci! Pintu dibuka.
  • False = Kamu tidak punya kunci! Penjaga diam saja dan kamu tidak bisa masuk.

Materi Bacaan 02

Menulis if dan else di Python

Pelajari bentuk kode, cara membaca kondisi, fungsi indentasi, dan bagaimana Python memilih satu dari dua jalan.

Sintaks Python
1

if

Memeriksa kondisi utama. Blok di bawahnya hanya dijalankan jika kondisi bernilai True.

2

else

Menjadi pilihan lain saat kondisi pada if bernilai False.

3

Indentasi

Spasi menjorok menandai perintah yang termasuk di dalam cabang if atau else.

Analogi Dua Jalan 🛣️

Jika if hanya punya satu pintu yang bisa terbuka atau tertutup, maka if-else adalah seperti persimpangan jalan.

  • Jalan pertama (if) diambil jika syarat terpenuhi.
  • Jalan kedua (else) otomatis diambil jika syarat pertama gagal. Kamu tidak akan pernah berjalan di kedua jalan tersebut sekaligus!

Contoh dua jalan keputusan

age = 15 if age >= 17: print("Boleh membuat KTP") else: print("Belum boleh membuat KTP")

Python hanya menjalankan satu cabang. Karena 15 >= 17 bernilai False, program melewati blok if dan menjalankan blok else.

Materi Bacaan 03

Multi-branch dengan if, elif, dan else

Multi-branch dipakai ketika program harus memilih satu hasil dari banyak kategori, misalnya kategori nilai, level, atau hari.

Banyak pilihan
1

Mulai dengan if

Cabang pertama memeriksa kondisi yang paling penting atau paling spesifik.

2

Lanjutkan dengan elif

Setiap elif hanya diperiksa jika semua kondisi sebelumnya salah.

3

Akhiri dengan else

else menangani semua nilai yang tidak cocok dengan cabang sebelumnya.

Pohon Keputusan (Decision Tree) 🌲

Kamu bisa membayangkan elif sebagai cabang-cabang pohon. Mulai dari batang utama (if), lalu kamu bisa menambahkan sebanyak apa pun cabang yang kamu butuhkan dengan elif. Jika tidak ada yang cocok, semuanya jatuh ke tanah (else).

Contoh kategori nilai

score = 82 if score >= 90: print("Sangat Baik") elif score >= 75: print("Baik") elif score >= 60: print("Cukup") else: print("Perlu latihan lagi")

Hasilnya adalah Baik. Nilai 82 gagal pada syarat pertama, berhasil pada syarat kedua, lalu Python berhenti memeriksa cabang berikutnya.

Mini Project: Smart Checker 👨‍🎓

Contoh lain yang sering dipakai adalah mengecek status penyelesaian tugas siswa bersama dengan nilai.

score = 82 task_done = True if not task_done: print("Selesaikan tugas dulu") elif score >= 85: print("Siap ikut kelas advance") elif score >= 70: print("Lanjut latihan berikutnya") else: print("Perlu belajar ulang")
Kesalahan umum: kondisi terlalu umum ditaruh di atas

Jika score >= 60 ditaruh paling atas, nilai 95 langsung masuk ke kategori “Cukup”. Susun rentang dari batas tertinggi ke batas terendah agar setiap kategori mendapat prioritas yang benar.

Materi Bacaan 04

Nested condition: pengecekan bertahap

Nested condition adalah kondisi di dalam kondisi. Pemeriksaan kedua baru dilakukan setelah pemeriksaan pertama berhasil.

Kondisi bersarang
A

Tahap pertama

Periksa syarat utama, misalnya apakah pengguna punya tiket atau sudah login.

B

Tahap kedua

Jika syarat utama terpenuhi, periksa syarat lanjutan seperti umur, role, atau jumlah koin.

C

Aksi berbeda per tahap

Nested cocok ketika tiap kegagalan perlu pesan atau tindakan yang berbeda.

Analogi Masuk Bioskop 🏢

Bayangkan kamu sedang pergi menonton bioskop:

  • Pintu Pertama: Penjaga mengecek tiket masuk. Kalau tidak punya tiket, kamu langsung dilarang masuk.
  • Pintu Kedua: Petugas di depan studio mengecek umurmu untuk disesuaikan dengan rating film.

Kamu tidak akan pernah sampai diperiksa di pintu kedua jika tidak lolos di pintu pertama!

Contoh pemeriksaan tiket dan umur

has_ticket = True age = 16 if has_ticket: if age >= 13: print("Boleh masuk studio") else: print("Umur belum cukup") else: print("Harus punya tiket dulu")

Perhatikan indentasinya. if age berada di dalam if has_ticket, sehingga umur hanya diperiksa ketika pengguna sudah memiliki tiket.

Kapan nested condition sebaiknya dipakai?
  • Ketika proses memang harus berlangsung secara berurutan.
  • Ketika setiap tahap memiliki pesan kegagalan yang berbeda.
  • Ketika kondisi kedua tidak relevan sebelum kondisi pertama terpenuhi.

Hindari terlalu banyak tingkat nested karena kode akan makin menjorok dan sulit dibaca.

Materi Bacaan 05

Menggabungkan kondisi dengan logical operators

and, or, dan not membantu kita mengecek beberapa syarat dalam satu ekspresi yang lebih ringkas.

Logika kombinasi
&

and: semua harus benar

Hasilnya True hanya jika setiap kondisi yang digabungkan bernilai True.

∨

or: salah satu cukup

Hasilnya True jika minimal satu kondisi bernilai True.

¬

not: membalik nilai

not True menjadi False dan not False menjadi True.

Contoh Sistem Diskon Toko 🏷️

Sebuah toko memberi diskon jika pelanggan mempunyai total belanja >= 100.000 dan adalah member, atau pelanggan punya kupon spesial.

total = 120000 is_member = True has_coupon = False if (total >= 100000 and is_member) or has_coupon: print("Mendapat diskon") else: print("Tidak mendapat diskon")

Tanda kurung () penting digunakan agar prioritas logika lebih mudah dipahami oleh programmer lain!

Contoh kondisi kompleks

score = 80 task_done = True remedial = False if score >= 75 and task_done and not remedial: print("Semua syarat terpenuhi")

score >= 75, task_done, dan not remedial semuanya True, jadi blok dijalankan.

Materi Bacaan 06

Financial literacy dengan Python

Gunakan data, conditional, logical operator, dan nested condition untuk membedakan kebutuhan, menyusun budget, serta menilai risiko keputusan keuangan.

Aplikasi nyata
1

Kebutuhan

Penting untuk hidup, belajar, atau aktivitas utama. Contoh: makanan, transportasi, buku, dan kuota belajar.

2

Keinginan

Disukai, tetapi dapat ditunda. Contoh: skin game, stiker, minuman tambahan, atau aksesori.

3

Keputusan berdasarkan data

Program mempertimbangkan kategori, harga, jumlah uang, sisa budget, dan rencana pengguna.

Analogi Uang Rp 50.000 ⚖️

Kalau kamu punya uang Rp 50.000, kamu harus memilih antara buku tulis (kebutuhan) dan stiker lucu (keinginan). Dalam Python, kita bisa membuat program yang mengecek kategori dan uang:

category = "kebutuhan" price = 15000 money = 50000 if category == "kebutuhan" and money >= price: print("Prioritaskan pembelian!") elif category == "keinginan" and money >= price: print("Boleh beli, tapi cek kebutuhan utama sudah terpenuhi belum") else: print("Uang belum cukup, cari alternatif lebih murah")

Langkah membuat budget

  1. Simpan jumlah uang yang tersedia.
  2. Bagi anggaran ke kategori makan, transportasi, sekolah, dan tabungan.
  3. Hitung total_budget dan remaining_money.
  4. Gunakan conditional untuk menentukan apakah budget aman, pas, atau terlalu besar.
total_money = 100000 food_budget = 40000 transport_budget = 20000 school_budget = 25000 saving_budget = 10000 total_budget = food_budget + transport_budget + school_budget + saving_budget remaining_money = total_money - total_budget if total_budget > total_money: print("Budget terlalu besar") elif remaining_money == 0: print("Budget pas, tidak ada sisa") else: print("Budget aman, masih ada sisa")

Menilai level risiko

LevelCiri utamaContoh keputusan
LowKebutuhan penting, uang cukup, dan masih ada cadangan.Membeli buku sekolah Rp25.000 dari Rp100.000.
MediumMasih dapat dipertimbangkan, tetapi hasil atau manfaat belum pasti.Memakai sebagian uang sebagai modal project kecil.
HighMenghabiskan hampir semua uang atau membeli keinginan mahal tanpa rencana.Menggunakan Rp95.000 untuk barang tren.

Semakin lengkap data yang diperiksa, semakin masuk akal rekomendasi program. Jangan hanya mengecek “uang cukup”, tetapi cek juga kategori barang dan sisa uang.

Contoh nested condition untuk rekomendasi risiko
remaining_money = money - price if category == "kebutuhan": if money >= price: risk_level = "Low" else: risk_level = "Medium" else: if money < price or remaining_money < 10000: risk_level = "High" elif price > money * 0.5: risk_level = "Medium" else: risk_level = "Low"
