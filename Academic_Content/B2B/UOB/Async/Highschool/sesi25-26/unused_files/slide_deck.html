Conditional Paling Sederhana: if

Di Python, conditional paling dasar menggunakan if.

age = 17

if age >= 17:
    print("Boleh membuat KTP")

Artinya:

Kalau age lebih besar atau sama dengan 17, tampilkan teks "Boleh membuat KTP".

Kalau kondisinya salah, program tidak menjalankan isi if.

Cara Membaca if 📖

if age >= 17:
    print("Boleh membuat KTP")

Cara membacanya:

"Jika umur lebih dari atau sama dengan 17, maka tampilkan pesan."

Bagian ini adalah kondisi:

age >= 17

Bagian ini adalah aksi yang dilakukan jika kondisi benar:

print("Boleh membuat KTP")

Kondisi Menghasilkan True atau False ⚖️

Setiap kondisi di Python akan menghasilkan salah satu dari dua jawaban:

True (Benar) atau False (Salah).

Contoh:

age = 17

print(age >= 17)

Hasilnya:

True

Karena 17 memang lebih besar atau sama dengan 17.

Kalau Kondisinya Salah? ❌

Contoh:

age = 15

if age >= 17:
    print("Boleh membuat KTP")

Program akan mengecek:
Apakah 15 >= 17?

Jawabannya:

False

Karena kondisinya salah, maka pesan di dalam if tidak akan muncul.

Menambahkan Pilihan Lain dengan else 🔀

Kadang program perlu melakukan sesuatu kalau kondisi salah.

Untuk itu, kita bisa menggunakan else.

age = 15

    print("Boleh membuat KTP")

    print("Belum boleh membuat KTP")

Artinya:

  • Kalau umur minimal 17, boleh membuat KTP.
  • Kalau tidak, belum boleh membuat KTP.

if else Itu Seperti Dua Jalan 🛣️

Bayangkan kamu sampai di persimpangan.

Kalau kondisi benar 👉 jalankan bagian if.

Kalau kondisi salah 👉 jalankan bagian else.

Multi-Branch Decision 🌳

Multi-branch decision adalah keputusan dengan banyak cabang pilihan.

Seperti memilih jalan:

  • Kalau nilai 90 ke atas 👉 Sangat Baik
  • Kalau nilai 75 ke atas 👉 Baik
  • Kalau nilai 60 ke atas 👉 Cukup
  • Kalau tidak masuk semua 👉 Perlu latihan lagi

Di Python, kita bisa membuatnya dengan gabungan:

if
elif
elif
else

Contoh Multi-Branch di Python 🐍

score = 82

if score >= 90:
    print("Sangat Baik")
elif score >= 75:
    print("Baik")
elif score >= 60:
    print("Cukup")
else:
    print("Perlu latihan lagi")

Hasilnya:

Baik

Karena 82 belum mencapai 90, tapi sudah mencapai 75.

Kenapa Urutan Kondisi Penting? ⚠️

Lihat contoh ini:

score = 95

if score >= 60:
    print("Cukup")
elif score >= 75:
    print("Baik")
elif score >= 90:
    print("Sangat Baik")

Hasilnya:

Cukup

Padahal skornya 95. Kenapa?

Karena Python membaca dari atas ke bawah. Saat menemukan kondisi pertama yang benar, Python langsung menjalankan bagian itu dan melewati sisanya.

Urutan yang Benar 🎯

Untuk kategori nilai, kita perlu mulai dari syarat tertinggi.

score = 95

if score >= 90:
    print("Sangat Baik")
elif score >= 75:
    print("Baik")
elif score >= 60:
    print("Cukup")
else:
    print("Perlu latihan lagi")

Hasilnya:

Sangat Baik

Jadi, dalam multi-branch decision, urutan kondisi harus dipikirkan dengan hati-hati.

Apa Itu Nested Condition? 🪆

Nested condition adalah conditional di dalam conditional.

Sederhananya:

Setelah program mengecek satu kondisi, program mengecek kondisi lain di dalamnya.

Contoh kehidupan nyata:

  • Kalau kamu punya tiket, kamu boleh masuk bioskop 🎟️.
  • Tapi setelah itu dicek lagi: Kalau umurmu cukup, baru kamu boleh menonton filmnya 🍿.

Jadi ada pengecekan di dalam pengecekan.

Analogi Nested Condition 🏢

Bayangkan kamu masuk ke gedung.

Pintu pertama mengecek:

Apakah kamu punya kartu akses? 💳

Kalau iya, kamu masuk ke pintu kedua.

Pintu kedua mengecek:

Apakah kamu punya izin masuk ruangan khusus? 🔐

Jadi, kamu harus lolos kondisi pertama dulu sebelum masuk ke kondisi kedua.

Itulah Nested Condition!

Contoh Nested Condition di Python 🐍

has_ticket = True
age = 16

if has_ticket:
    if age >= 13:
        print("Boleh masuk studio")
    else:
        print("Umur belum cukup")
else:
    print("Harus punya tiket dulu")

Program pertama mengecek: Apakah punya tiket?

Kalau iya, baru program mengecek: Apakah umur minimal 13?

Cara Membaca Nested Condition 🧐

if has_ticket:
    if age >= 13:
        print("Boleh masuk studio")

Cara membacanya:

"Kalau punya tiket, cek lagi umurnya."

Perhatikan bahwa if yang kedua letaknya lebih menjorok ke dalam (di-indentasi)!

Nested Condition di Program Game 🎮

Contoh dalam game: Pemain hanya bisa masuk level bonus jika:

  • Pemain sudah menyelesaikan level 1.
  • Dan pemain punya minimal 100 koin.

Dengan nested condition:

level_complete = True
coin = 120

if level_complete:
    if coin >= 100:
        print("Masuk level bonus")
    else:
        print("Koin belum cukup")
else:
    print("Selesaikan level dulu")

Masalah Nested Condition 😵‍💫

Nested condition memang berguna. Tapi kalau terlalu banyak, kodenya bisa terlihat menjorok terlalu dalam dan sulit dibaca.

Contoh:

if login:
    if verified:
        if balance >= price:
            print("Pembelian berhasil")

Kode seperti ini masih bisa dibaca, tapi kalau terlalu panjang bisa membingungkan.

Karena itu, kita perlu belajar Logical Operators! ✨

Menggabungkan Kondisi (Logical Operators) 🔗

Kadang kita ingin mengecek beberapa kondisi sekaligus dalam satu baris, tanpa perlu nested condition.

Di Python, kita bisa memakai:

and or not

Ketiganya disebut Logical Operators.

Logical operators membantu program membuat keputusan yang lebih kompleks dengan cara yang rapi!

Operator and 🤝

and digunakan kalau semua kondisi harus benar.

Contoh kehidupan nyata: Kamu boleh ikut lomba kalau:

  • Sudah daftar.
  • Sudah membayar.
  • Sudah mengumpulkan karya.

Kalau salah satu saja belum, kamu belum boleh ikut.

Contoh and di Python 🐍

registered = True
paid = True
submitted = True

if registered and paid and submitted:
    print("Boleh ikut lomba")
else:
    print("Belum bisa ikut lomba")

Program akan menampilkan pesan berhasil hanya kalau ketiga kondisi tersebut bernilai True.

Latihan: Logika and 🧩

Instruksi: Lengkapi program beasiswa berikut agar berjalan dengan benar!

  • Syarat Beasiswa: Nilai harus lebih dari 80 dan aktif organisasi bernilai True.
  • Ketik operator logika dan variabel kondisi yang tepat di dalam kotak.
nilai = 85
aktif_organisasi = True

if nilai > 80 :
    print("Selamat, kamu dapat beasiswa!")
else:
    print("Belum memenuhi syarat.")

Operator or 🤷‍♂️

or digunakan kalau cukup salah satu kondisi yang benar.

Contoh kehidupan nyata: Kamu bisa login kalau menggunakan:

  • Email 📧, atau
  • Username 👤, atau
  • Nomor HP 📱.

Tidak harus semuanya. Salah satu saja cukup!

Contoh or di Python 🐍

use_email = False
use_username = True
use_phone = False

if use_email or use_username or use_phone:
    print("Bisa login")
else:
    print("Pilih salah satu cara login")

Hasilnya:

Bisa login

Kenapa? Karena salah satu kondisi, yaitu use_username, bernilai True.

Latihan: Logika or 🧩

Instruksi: Lengkapi program diskon berikut!

  • Syarat Diskon: Pembeli adalah pelajar atau memiliki kupon diskon.
  • Ketik operator logika dan variabel kondisi yang tepat di dalam kotak.
status = "umum"
ada_kupon = True

if status == "pelajar" :
    print("Hore! Kamu dapat diskon.")
else:
    print("Harga normal.")

Operator not 🔄

not digunakan untuk membalik nilai kondisi.
  • Kalau awalnya True, menjadi False.
  • Kalau awalnya False, menjadi True.

Contoh kehidupan nyata:

Kalau belum hujan, kita bisa bermain di luar.
is_raining = False

if not is_raining:
    print("Boleh bermain di luar")

Artinya: Kalau tidak sedang hujan, boleh bermain di luar.

Menggabungkan 3+ Kondisi 🤯

Sekarang kita bisa membuat kondisi yang lebih lengkap!

Contoh: Siswa bisa ikut kelas lanjutan jika:

  • Nilai minimal 75.
  • Sudah menyelesaikan tugas.
  • Tidak sedang remedial.
score = 80
task_done = True
remedial = False

if score >= 75 and task_done and not remedial:
    print("Boleh ikut kelas lanjutan")
else:
    print("Belum boleh ikut kelas lanjutan")

Cara Membaca Kondisi Kompleks 🧐

score >= 75 and task_done and not remedial

Bacanya:

  • Nilai harus minimal 75. (True)
  • Tugas harus sudah selesai. (True)
  • Siswa tidak sedang remedial. (True)

Karena memakai and, semua syarat harus benar.

Kalau ada satu saja yang salah, hasil akhirnya menjadi False.

Prioritas Operator Logika 🚦

Python punya urutan dalam membaca operator logika.

  1. not
  2. and
  3. or

Artinya, Python akan mengecek not dulu, lalu and, lalu or.

Contoh:

True or False and False

Python membaca bagian and dulu: False and False (hasilnya False).

Lalu: True or False.

Hasil akhirnya: True.

Gunakan Tanda Kurung 📝

Walaupun Python punya urutan sendiri, kita bisa memakai tanda kurung agar kode lebih mudah dibaca.

Contoh:

if (score >= 75 and task_done) or has_bonus:
    print("Lolos")

Dengan tanda kurung, kita tahu bahwa Python harus mengecek ini dulu:

score >= 75 and task_done

Lalu hasilnya dibandingkan dengan has_bonus.

Tanda kurung membuat kondisi lebih jelas!

  • Instruksi: Tuliskan logika yang tepat di dalam kotak. Jangan lupa gunakan tanda kurung agar pengecekan "admin atau premium" didahulukan!
password_ok = True

if :

Truth Table and 🤝

and hanya menghasilkan True kalau semua kondisi benar.

Kondisi A Kondisi B A and B
True True True
True False False
False True False
False False False
Cara mengingat: and itu seperti "semua harus lengkap".

Truth Table or 🤷‍♂️

or menghasilkan True kalau minimal satu kondisi benar.

Kondisi A Kondisi B A or B
True True True
True False True
False True True
False False False
Cara mengingat: or itu seperti "salah satu saja cukup".

Truth Table not 🔄

not berfungsi untuk membalik nilai.

Kondisi not Kondisi
True False
False True
Cara mengingat: not itu seperti tombol pembalik.

Mengevaluasi Kondisi Kompleks 🕵️‍♂️

Contoh:

score = 80
task_done = True
remedial = False

score >= 75 and task_done and not remedial

Kita cek satu per satu:

  • score >= 75 👉 True
  • task_done 👉 True
  • not remedial 👉 True (karena remedial itu False)

Jadi: True and True and True.

Hasil akhirnya: True

Nested Condition vs Logical Operators 🥊

Kode nested yang menjorok terlalu jauh:

if level_complete:
    if coin >= 100:
        if not game_over:
            print("Masuk level bonus")

Bisa disederhanakan menjadi:

if level_complete and coin >= 100 and not game_over:
    print("Masuk level bonus")
Kode kedua lebih pendek dan lebih mudah dibaca!

Kapan Pakai Nested Condition? 🧐

Pakai nested condition kalau pengecekan memang bertahap.

Contoh:

  1. Pertama cek login.
  2. Kalau login benar, baru cek role user.
  3. Kalau role admin, tampilkan menu admin.
if login:
    if role == "admin":
        print("Tampilkan menu admin")
    else:
        print("Tampilkan menu user")
else:
    print("Silakan login dulu")

Nested cocok kalau setiap tahap punya pesan atau aksi yang berbeda.

Kapan Pakai Logical Operators? 💡

Pakai logical operators kalau beberapa syarat bisa dicek sekaligus.

Contoh:

if login and role == "admin":
    print("Tampilkan menu admin")

Ini cocok kalau program hanya perlu tahu:

Apakah semua syarat terpenuhi? Kalau iya, jalankan aksi.

Latihan: Nested ke Logical 🛠️

Ubah kode bercabang yang ribet ini menjadi satu baris kondisi dengan operator logika!

# Kode Awal (Ribet):
if hari == "sabtu":
    if cuaca == "cerah":
        print("Main bola")
# Kode Barumu (Singkat):
if hari == "sabtu" :
    print("Main bola")

Latihan: Logical ke Multi-Branch 🌲

Kode di bawah menggunakan banyak if secara terpisah. Ubahlah menjadi lebih rapi menggunakan struktur Multi-Branch!

# Kode Awal (Kurang efisien):
if score > 80:
    print("A")
if score > 60 and score <= 80:
    print("B")
if score <= 60:
    print("C")
# Kode Barumu (Rapi):
if score > 80:
    print("A")
score > 60:
    print("B")
:
    print("C")

Contoh Kasus: Sistem Diskon 🏷️

Sebuah toko memberi diskon jika pelanggan:

  • Total belanja minimal 100.000 dan pelanggan adalah member.
  • Atau pelanggan punya kupon.
total = 120000
is_member = True
has_coupon = False

if (total >= 100000 and is_member) or has_coupon:
    print("Mendapat diskon")
else:
    print("Tidak mendapat diskon")

Tanda kurung penting supaya kondisi mudah dipahami!

Membaca Sistem Diskon 🕵️‍♂️

(total >= 100000 and is_member) or has_coupon

Artinya:

Pelanggan mendapat diskon jika:

  • Total belanja minimal 100.000 dan dia member.
Atau:
  • Pelanggan punya kupon.

Jadi ada dua jalur untuk mendapat diskon.

Contoh Kasus: Akses Aplikasi 📱

Seseorang boleh masuk aplikasi jika:

  • Username benar.
  • Password benar.
  • Akun tidak diblokir.
username_correct = True
password_correct = True
blocked = False

if username_correct and password_correct and not blocked:
    print("Login berhasil")
else:
    print("Login gagal")
Karena memakai and, semua syarat utama harus benar. Dan karena ada not blocked, akun harus dalam keadaan tidak diblokir.

Kasus: Rekomendasi Aktivitas 🏃‍♂️

Program rekomendasi aktivitas berdasarkan cuaca dan waktu.

weather = "cerah"
time = "pagi"

if weather == "cerah" and time == "pagi":
    print("Jogging di taman")
elif weather == "cerah" and time == "siang":
    print("Minum es dan istirahat")
elif weather == "hujan":
    print("Aktivitas di rumah")
else:
    print("Cek kondisi lagi")

Ini contoh multi-branch decision dengan kondisi gabungan.

Kesalahan Umum 1: Lupa Tanda == ⚠️

Dalam Python:

  • = dipakai untuk menyimpan data.
  • == dipakai untuk membandingkan data.

Salah (Error):

if password = "12345":
    print("Login")

Benar:

if password == "12345":
    print("Login")

Kesalahan Umum 2: Kondisi Panjang 🥵

Contoh kondisi yang terlalu panjang:

if score >= 75 and task_done == True and remedial == False and attendance >= 80:
    print("Lolos")

Bisa dibuat lebih rapi:

passed_score = score >= 75
completed_task = task_done
not_remedial = not remedial
good_attendance = attendance >= 80

if passed_score and completed_task and not_remedial and good_attendance:
    print("Lolos")
Kode lebih panjang sedikit, tapi jauh lebih mudah dibaca.

Kesalahan Umum 3: Salah Urutan elif 📉

Salah (Error Logika):

score = 92

if score >= 60:
    print("Cukup")
elif score >= 75:
    print("Baik")
elif score >= 90:
    print("Sangat Baik")

Benar:

score = 92

if score >= 90:
    print("Sangat Baik")
elif score >= 75:
    print("Baik")
elif score >= 60:
    print("Cukup")

Financial literacy adalah kemampuan memahami dan mengatur uang dengan bijak.

Bagian 1

Kebutuhan vs Keinginan + Python Conditionals

Apa Itu Kebutuhan? 🛒

Kebutuhan adalah sesuatu yang penting untuk hidup, belajar, atau menjalankan aktivitas utama.

Contoh kebutuhan:

  • 🍽️ Makanan
  • 💧 Air minum
  • 📚 Buku sekolah
  • ✏️ Alat tulis
  • 🚌 Transportasi
  • 🌐 Kuota internet belajar

Kebutuhan biasanya harus diprioritaskan dulu.

Apa Itu Keinginan? 🎮

Keinginan adalah sesuatu yang kita suka, tapi belum tentu harus dibeli sekarang.

Contoh keinginan:

  • 🍟 Snack tambahan
  • 🎮 Skin game
  • 🔑 Gantungan kunci lucu
  • 🧋 Minuman kekinian
  • 👾 Stiker
  • 📱 Aksesoris HP

Keinginan boleh dibeli, tapi sebaiknya setelah kebutuhan utama aman.

Analogi Sederhana ⚖️

Bayangkan kamu punya uang Rp 50.000.

  • Buku tulis: Rp 15.000
  • Makan siang: Rp 20.000
  • Stiker lucu: Rp 10.000
  • Minuman manis: Rp 15.000

Kalau uang terbatas, kita perlu memilih.
Mana yang harus dibeli dulu? Mana yang bisa ditunda?

Di sinilah kita belajar membedakan kebutuhan dan keinginan.

Dari Keputusan ke Kondisi 💻

Dalam kehidupan nyata:

  • Kalau barang adalah kebutuhan, prioritaskan.
  • Kalau barang adalah keinginan, cek dulu sisa uang.
  • Kalau uang tidak cukup, tunda pembelian.

Dalam Python, ini bisa dibuat seperti:

if category == "kebutuhan":
    print("Prioritaskan pembelian")
elif category == "keinginan":
    print("Cek budget dulu")
else:
    print("Kategori belum jelas")

Tambahan Materi

Risiko Finansial + Nested Condition ⚠️

Apa Itu Risiko Finansial? 🤔

Risiko finansial adalah kemungkinan uang kita berkurang, habis, atau tidak menghasilkan sesuai harapan.

Contoh sederhana:

  • Kalau kita menyimpan uang di tempat aman, risikonya kecil.
  • Kalau kita memakai uang untuk sesuatu yang hasilnya belum pasti, risikonya lebih besar.

Kita perlu bertanya: Apakah keputusan ini aman? Apakah ada kemungkinan rugi? Apakah aku siap kalau hasilnya tidak sesuai harapan?

Risiko Seperti Level Kesulitan 🎮

Bayangkan kamu bermain game. Ada level: Easy, Normal, Hard.

Dalam keuangan, risiko juga bisa dibayangkan seperti level:

Low Risk
Medium Risk
High Risk

Semakin tinggi risikonya, semakin besar kemungkinan hasilnya berubah-ubah.

Low Risk 🟢

Low risk artinya risiko rendah. Biasanya lebih aman, tetapi hasilnya juga tidak terlalu besar.

Contoh sederhana:

  • Menabung
  • Menyimpan uang untuk kebutuhan penting
  • Membeli barang yang benar-benar dibutuhkan
  • Membuat dana cadangan

Cocok untuk uang yang tidak boleh hilang atau uang yang akan dipakai dalam waktu dekat.

Medium Risk 🟡

Medium risk artinya risiko sedang. Masih bisa dipertimbangkan, tetapi perlu lebih hati-hati.

Contoh sederhana:

  • Membeli barang untuk dijual lagi
  • Mengikuti project kecil yang butuh modal
  • Membeli perlengkapan untuk membuat produk
  • Mencoba ide usaha kecil

Cocok kalau kita sudah punya rencana dan masih punya sisa uang aman.

High Risk 🔴

High risk artinya risiko tinggi. Kemungkinan untung bisa ada, tetapi kemungkinan rugi juga lebih besar.

Contoh sederhana:

  • Menggunakan hampir semua uang untuk satu hal
  • Membeli barang mahal yang belum tentu berguna
  • Ikut tren hanya karena takut ketinggalan
  • Mengeluarkan modal tanpa rencana

High risk perlu dipikirkan dengan sangat hati-hati.

Cara Sederhana Menilai Risiko 📏

Kita bisa menilai risiko dengan beberapa pertanyaan:

  1. Apakah uang ini akan dipakai untuk kebutuhan penting?
  2. Apakah uang yang dipakai terlalu besar?
  3. Apakah masih ada sisa uang setelah keputusan ini?
  4. Apakah keputusan ini punya hasil yang belum pasti?
  5. Apakah kita punya rencana cadangan?

Semakin banyak jawaban yang membuat kita ragu, semakin tinggi risikonya.

Contoh Penilaian Risiko 📝

Contoh 1: Beli buku sekolah Rp25.000 dari uang Rp100.000.
Risiko: Low risk (Kebutuhan & uang sisa).
Contoh 2: Beli bahan jualan Rp60.000 dari uang Rp100.000.
Risiko: Medium risk (Peluang berguna tapi belum pasti).
Contoh 3: Pakai Rp95.000 untuk barang tren.
Risiko: High risk (Hampir semua uang habis untuk keinginan).

Mengubah Risiko Jadi Kondisi Python 🐍

Dalam Python, kita bisa membuat program yang mengecek level risiko. Contoh data yang dipakai:

money = 100000
price = 25000
category = "kebutuhan"
has_plan = True

Program bisa mengecek:

  • Apakah barang kebutuhan atau keinginan?
  • Apakah harga terlalu besar?
  • Apakah masih ada sisa uang?
  • Apakah pengguna punya rencana?

Kenapa Pakai Nested Condition? 🧐

Risiko sering tidak bisa dicek hanya dengan satu pertanyaan. Pengecekan dilakukan bertahap:

  • Pertama, cek kategorinya.
  • Kalau kebutuhan, cek apakah uang cukup.
  • Kalau keinginan, cek apakah sisa uang masih aman.
  • Kalau untuk usaha, cek apakah ada rencana.

Karena pengecekannya bertahap, kita bisa memakai nested condition.

Program Simple: Cek Risiko Berdasarkan Kategori 🚦

category = "kebutuhan"
money = 100000
price = 25000

if category == "kebutuhan":
    if money >= price:
        print("Risiko: Low")
        print("Alasan: Barang ini kebutuhan dan uang cukup.")
    else:
        print("Risiko: Medium")
        print("Alasan: Barang ini kebutuhan, tapi uang belum cukup.")
else:
    if price > money * 0.5:
        print("Risiko: High")
        print("Alasan: Harga barang terlalu besar untuk keinginan.")
    else:
        print("Risiko: Medium")
        print("Alasan: Barang ini keinginan, jadi perlu dipertimbangkan.")

Cara Membaca Program Risiko 📖

Bagian pertama:

if category == "kebutuhan":

Program mengecek apakah barang termasuk kebutuhan. Kalau iya, program masuk ke pengecekan berikutnya:

if money >= price:

Artinya: Kalau uang cukup, risikonya rendah. Tapi kalau uang belum cukup, risikonya naik menjadi medium.

Nested Condition untuk Rekomendasi Multi-Level 📉

Sekarang kita buat rekomendasi dengan 3 level risiko.

money = 100000
price = 60000
category = "keinginan"

remaining_money = money - price

if category == "kebutuhan":
    if money >= price:
        print("Risiko: Low")
        print("Rekomendasi: Boleh dibeli karena ini kebutuhan.")
    else:
        print("Risiko: Medium")
        print("Rekomendasi: Cari alternatif yang lebih murah.")
else:
    if remaining_money < 10000:
        print("Risiko: High")
        print("Rekomendasi: Tunda dulu karena sisa uang terlalu sedikit.")
    elif price > money * 0.5:
        print("Risiko: Medium")
        print("Rekomendasi: Pertimbangkan lagi karena harganya cukup besar.")
    else:
        print("Risiko: Low")
        print("Rekomendasi: Boleh dipertimbangkan jika kebutuhan sudah aman.")

Menambahkan Faktor Rencana 📝

Untuk keputusan yang berhubungan dengan usaha kecil atau project, kita bisa menambahkan variable:

has_plan = True

Artinya: Apakah pengguna punya rencana yang jelas? Contoh rencana:

  • Tahu barangnya untuk apa
  • Tahu perkiraan biaya
  • Tahu target hasil
  • Tahu risiko kalau tidak berhasil

Program Risiko dengan Rencana 🛠️

money = 100000
price = 60000
category = "usaha"
has_plan = True

remaining_money = money - price

if category == "usaha":
    if has_plan:
        if remaining_money >= 20000:
            print("Risiko: Medium")
            print("Alasan: Ada rencana dan masih ada sisa uang aman.")
        else:
            print("Risiko: High")
            print("Alasan: Ada rencana, tapi sisa uang terlalu sedikit.")
    else:
        print("Risiko: High")
        print("Alasan: Belum ada rencana yang jelas.")
else:
    print("Gunakan pengecekan kebutuhan atau keinginan.")

Ini contoh nested condition karena pengecekannya bertahap: Pertama cek kategori, lalu cek apakah punya rencana, lalu cek sisa uang.

Membuat Alasan Logis dari Program 🧠

Program yang baik tidak hanya memberi hasil:

print("Risiko: High")

Tapi juga memberi alasan:

print("Alasan: Sisa uang terlalu sedikit setelah membeli barang.")

Kenapa alasan penting? Agar pengguna tahu mengapa program memberi rekomendasi tersebut. Dalam financial literacy, keputusan harus bisa dijelaskan, bukan hanya ditebak.

Contoh Output Program Risiko 💻

Contoh input/data:

money = 100000
price = 80000
category = "keinginan"

Output:

Risiko: High
Rekomendasi: Tunda dulu karena sisa uang terlalu sedikit.
Alasan: Barang ini keinginan dan memakai sebagian besar uang.

Keputusan ini masuk akal karena uang tersisa hanya Rp20.000.
